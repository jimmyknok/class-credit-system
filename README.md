# 班级积分系统 / System Requirements Overview

基于 Figma Make 导出的 React 项目，根目录为主应用。原 HTML 版本已移至 `legacy/` 目录。

## 版本线维护

- `main`: 2.x 主线（React + Vite，多框架集成）
- `html-v3`: 3.x 主线（单文件 HTML 维护线）

单文件版本代码位于 `single-html/`，发布 tag 使用 `v3.*`。

## 项目结构

```
class-credit-system/     # 主项目（React + Vite）
├── src/                 # 源码
├── legacy/              # 原 HTML 班级积分系统（备份）
├── single-html/         # v3 单文件 HTML 维护目录（html-v3 分支）
└── ...
```

## 安装与运行

```bash
# 安装依赖（需确保代理正确，项目根目录 .npmrc 已配置端口 7897）
npm install

# 启动开发服务器
npm run dev
```

开发服务器默认在 `http://localhost:5173` 启动。

## 构建

```bash
npm run build
```

## npm run dev 后的预期页面

1. **总览页 (/)**
   - 班级积分统计卡片（学生总数、平均积分、最高积分、总记录数）
   - 积分排行榜柱状图
   - 小组排行
   - 最新积分记录列表
   - 快捷操作按钮（加减积分、随机点名、课堂计时、积分商店）

2. **系统需求页 (/system-requirements)**
   - 标题「系统需求概览」
   - 筛选区：按优先级（全部/高/中/低）和状态（全部/草稿/审核中/已通过/已实现）过滤
   - 统计卡片：总需求数、高优先级数、已实现数、审核中数
   - 表格/卡片视图切换
   - **表格视图**：可展开行显示完整描述，列含 ID、描述、优先级（Badge）、状态
   - **卡片视图**：每项为 Card，含 ID、描述、优先级 Badge、状态
   - 10 条 mock 需求数据（REQ-001 至 REQ-010）

3. **其他页面**
   - 学生管理、小组管理、积分操作、积分规则、积分记录、积分商店、课堂工具、数据管理

## 响应式

- 移动端：侧边栏折叠为汉堡菜单，统计与筛选纵向排列
- 平板/桌面：侧边栏常驻，网格与表格自适应
